# Seed Data Policy v1

## Purpose

Seed data may be used for development and testing only.

Seed data must never contain real client information, real legal facts, real offense details, real victim information, or realistic identifiable case material.

## Allowed Seed Data

Allowed:

- fictional participant names
- fictional emails using example domains
- generic relationship statuses
- generic check-in responses
- generic weekly reflections that avoid offense details
- fictional clinician users
- generic participation summaries

## Prohibited Seed Data

Do not include:

- real client names
- real therapist names unless approved
- real attorney names
- real court information
- real legal facts
- offense narratives
- victim names
- child names
- investigative facts
- police facts
- illegal content descriptions
- realistic CSAM/CSM details
- real case patterns
- protected health information
- personally identifiable information from actual clients

## Example Participant Names

Use fictional names such as:

- Michael Example
- James Sample
- David Test
- Robert Demo

## Example Domains

Use:

- `example.com`
- `test.local`
- `demo.local`

## Sample Reflection Rule

Sample reflections should focus on:

- shame
- isolation
- boundaries
- support contact
- therapist follow-up
- digital integrity

They must not describe illegal behavior or alleged facts.
